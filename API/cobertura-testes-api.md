# Cobertura de Testes de API Rest

Cobertura de testes com base em requisitos, cobertura com base em riscos, cobertura com base em códigos, etc;

Abordagem Martin Lopes: escreveu artigo propondo cobertura de testes em API REST: que seria cobrir a interface da API REST.

A proposta se baseia em analisar a API REST através dos inputs e outputs. Exs:

    Inputs: A quantidade de caminhos (path) que existem na API REST.
    Operation coverege: mescla entre o uso do método junto com o endpoint.
    Parameter coverege: testes para cobrir os parâmetros escritos no envio deles.
    Parameter value coverege: testes de valor enviado no parâmetro.
